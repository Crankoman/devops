## 2. Какая альтернатива без pipe команде grep <some_string> <some_file> | wc -l?

<-
`grep <some_string> <some_file>` будет искать вхождение подстроки в файле и выводить строки где данная подстрока встречается. `wc -l` подсчитывает количество строк(с переводом каретки) 

Как альтернативу можно использовать `grep <some_string> <some_file> -c` 

Параметр `-c` -count который, так же вернет количество строк с вхождением

## 9. Что выведет команда `cat /proc/$$/environ`? Как еще можно получить аналогичный по содержанию вывод?

<- 
Судя по `man proc` строка ***191*** `/proc/[pid]/environ`  файл содержит начальную среду(набор переменных окружения), которые была установлены, когда текущая исполняемая программа была запущена через execve.

Аналогичный по содержимому вывод мжно получить через ` ps e -o command -p $$` также будут перечислеты переменные процесса, 
`-o command` выводим только столбец COMMAND
Параметр `e` выводим переменные после команды

Похожий вывод дает также `env`

## 10. Используя man, опишите что доступно по адресам `/proc/<PID>/cmdline`, `/proc/<PID>/exe`.

<- 
`man proc` 

`/proc/[pid]/cmdline` строка ***146*** Этот доступный только для чтения файл содержит полную командную строку для процесса с указанным `<PID>`, если этот процесс не является zombi.

    cat /proc/$$/cmdline
    -bashv

`/proc/<PID>/exe` строка ***98*** В Linux 2.2 и более поздних версиях этот файл представляет собой символическую ссылку, содержащую фактический путь к файлу выполняемой команды для процесса с указанным `<PID>`. попытка открыть его откроет исполняемый файл. 
попытка выполнить `cat proc/[pid]/exe` выведет содержимое файла

## 11. Узнайте, какую наиболее старшую версию набора инструкций SSE поддерживает ваш процессор с помощью /proc/cpuinfo.

<- 
С помощью `cat` содержимое /proc/cpuinfo, через pipe передаем в `grep` чтобы нати вхождение

В `grep` используем парметры -i -  регистронезависимый поиск, -o - показывать только часть строки, совпадающей с шаблоном, шаблон регулярного выражения '\w*' - любое количество цифр, букв или знаков подчеркивания передаем через pipe в `sort`

В `sort` используем парметры -r - обратная сортировка(от большего к меньшему), -u - убрать дубликаты



    cat /proc/cpuinfo | grep -io 'sse\w*' | sort -r -u
    sse4a
    sse4_2
    sse4_1
    sse3
    sse2
    sse

Ответ:  sse4a ну или sse4_2 (непонятно что старше)
